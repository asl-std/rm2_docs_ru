Книги рецептов

Этот плагин автоматически генерирует письменные элементы книги с написанными в них рецептами.
Книги настраиваются и хранятся в /plugins/RecipeManager/books/ как файлы .yml.

Вы можете создавать новые книги, просто создав новый файл .yml в этой папке, но, в отличие от рецептов, эта папка не поддерживает подпапки.

Имя файла книги определяет идентификатор книги, который будет использоваться для идентификации книг, даже если они переименованы.
Это полезно при редактировании уже имеющихся у игроков книг, которые плагин будет обновлять автоматически. Если вы измените имя файла, их старые книжные элементы больше не будут обновляться.

В файле .yml книги могут быть:

title: The Book Title author: RecipeManager description: Book description written on the first page # настройки книги и их значения по умолчанию settings: pervolume: 50 cover: true contents: true end: true customend: '' # список рецептов для автоматического распределения по объемам recipes: - list of - recipe names or - + special arguments # или вы можете напрямую определять отдельные объемы volume 1: - recipe name - ... volume 2: - recipe name - etc ... volume ... # это, кстати, комментарии, но они не будут сохранены, так как плагин перезапишет файлы.

Учтите, что книга не может содержать один и тот же рецепт дважды.

'title' (необязательно, имя файла по умолчанию) не должен быть длиннее 64 символов, поддерживает цвета.

'author' (необязательный, по умолчанию "RecipeManager"), не должен быть длиннее 64 символов, поддерживает цвета.

'description' (необязательно, по умолчанию ничего) не должен быть длиннее 100 символов, поддерживает цвета.

'pervolume (необязательно, по умолчанию 50) определяет максимальное количество рецептов на том, которое выделяет узел 'recipes'
Это не ограничивает вас в добавлении большего количества рецептов в объемах.

'cover', 'contents' и 'end' (необязательно, по умолчанию все true) переключить, если для книги создаются обложка, содержание или конечные страницы.
Вы можете переключать их в любое время, но вам нужно перезагрузить книги, чтобы они подействовали.

'customend' устанавливает собственный текст для последней страницы, работает, только если 'end' истинно. Поддерживает цвета и \ n для новых строк.

Затем вы можете использовать узел 'recipes' чтобы перечислить рецепты, которые вы хотите добавить, или использовать специальные аргументы (см. Ниже).
После определения 'recipes' определен и книга загружена, узел будет удален и автоматически сгенерирует тома в соответствии с ограничением количества рецептов на том.

Вы можете снова добавить узел 'recipes' чтобы добавлять рецепты к существующим томам автоматически, или вы можете просто добавить их непосредственно в нужные вам тома.

Чтобы добавить рецепты в книгу, дайте ей собственное имя или используйте @addtobook flag.

Имена рецептов должны быть уникальными, если они определены в их файлах рецептов, но если у рецептов нет определенного имени, они будут иметь уникально сгенерированное имя.
Это сгенерированное имя содержит тип рецепта, его ингредиент (s) и его результат (s).

Вы также можете использовать специальные аргументы, чтобы добавить сразу несколько рецептов, доступные специальные аргументы:

+existing <all/workbench/smelt/fuel>
Он заменяется существующими рецептами, соответствующими указанному типу.
Использование 'all' добавит верстак, рецепты плавки и топлива по порядку.

+custom <all/workbench/smelt/fuel>
This is replaced by custom recipes (added with this plugin) matching the type specified.
Использование 'all' добавит верстак, рецепты плавки и топлива по порядку.

+file </path/to/file.ext>
+folder </path/to/folder>
Они заменяются рецептами, определенными в заданном файле рецептов, или файлами рецептов из заданной папки.
Пути указаны относительно '/plugins/RecipeManager/recipes' так определяющий '/' поскольку путь будет означать корень папки 'recipes'.

После загрузки книги сервером эти аргументы будут заменены в файле найденными рецептами.

Если вы используете какой-либо +argument в узле 'recipes' они будут автоматически распределены по объемам, как определено лимитом 'recipes-per-volume'. Однако, если вы добавляете их непосредственно в том, они будут добавлены непосредственно в этот том, потому что вы не связаны лимитом объема, а только узлом 'recipes'.

Кроме того, когда файл книги загружен, к нему будет добавлен комментарий заголовка, указывающий дату / время последнего чтения.

После добавления рецептов вы можете просто перемещать их или удалять, если хотите.

Теперь ваша книга должна быть готова.

тобы увидеть доступные книги рецептов, введите 'rmbooks' в консоли сервера или в качестве оператора.

Вы можете использовать @getrecipebook отметьте в рецепте, чтобы создать книгу рецептов.

Кроме того, вы можете получить книги с помощью команды 'rmgetbook' доступной операторам.

Пример книги: 'swords.yml'

title: Stuff! description: Random things. recipes: - flaming sword of doom - oblivion ice axe - shaped 3x2 (76 331:0 76 / 1 1 1) diode - shaped 1x3 (5 / 5 / 280:0) wooden_sword - shaped 3x3 (5 5 5 / 0 280:0 0 / 0 280:0 0) wood_pickaxe - some non-existent recipe name here - shaped 1x3 (4 / 4 / 280:0) stone_sword

Что после использования команды, rmreload или rmreloadbooks, содержимое книги будет преобразовано в:

# Конфигурация книги рецептов (last loaded at May 2, 2013 12:19:45 PM) # Прочтите файл 'recipe books.html' чтобы узнать, как настраивать книги. title: Stuff! description: Random things. volume 1: - flaming sword of doom - oblivion ice axe - shaped 3x2 (76 331:0 76 / 1 1 1) diode - shaped 1x3 (5 / 5 / 280:0) wooden_sword - shaped 3x3 (5 5 5 / 0 280:0 0 / 0 280:0 0) wood_pickaxe - shaped 1x3 (4 / 4 / 280:0) stone_sword

ПРИМЕЧАНИЕ Вы не должны писать автоматически сгенерированные названия рецептов вручную!
Как было сказано ранее, вы можете дать им собственное имя или использовать @addtobook
Для существующих рецептов вы можете использовать специальный аргумент +existing чтобы получить все имена, а затем удалить те, которые вам не нужны.

После этого вы все еще можете изменить его содержимое, переставить рецепты, удалить рецепты, изменить его название, описание, все, что захотите!
Только убедитесь, что вы не переименовали файл, если у людей уже есть копии книги, в этом случае их книги не будут обновляться!
