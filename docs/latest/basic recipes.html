Вступление

Как сделать файл рецепта

Сначала перейдите к <расположение вашего сервера>\plugins\RecipeManager\recipes папку и создайте текстовый файл (.txt extension).

Затем вы можете открыть этот файл в текстовом редакторе и начать писать рецепты.

Файлы рецептов

Вы можете иметь столько файлов рецептов и рецептов в файле, сколько захотите. Вы также можете создавать папки для организации.

Комментарии и интервалы

Поддерживаемые комментарии по умолчанию: //, / * * / и #

// Это комментарий # Это тоже комментарий /* Многострочные комментарии также поддерживаются */

Текст в файлах рецептов может иметь любой регистр букв и может содержать любое количество пробелов, табуляции и новых строк.

// Файлы рецептов поддерживают любую оболочку craft cobblestone = cobblestone // Добавляйте любое количество новых строк между рецептами CRAFT COBBLESTONE = COBBLESTONE // Пробел / вкладки в начале тоже подойдут CrAfT cobbleSTONE = COBBLEstone // Расстояние не обязательно должно быть одинаковым. craft cobblestone = cobblestone

Обязательный и необязательный

Обязательные поля обведены < >
Необязательные поля заключены в [ ]

craft [recipe name] // Название рецепта необязательно, его можно пропустить. <material> // Этот материал обязателен = <material> // Также необходим результат

// Не печатайте <, >, [ or ] craft [my recipe] <cobblestone> = <cobblestone>

// Правильный способ написать рецепт выше // Обратите внимание, что символы за пределами <, >, [ and ] постоянны и никогда не меняются craft my recipe // 'craft' is a constant here cobblestone = cobblestone

Специальные символы

ПРИМЕЧАНИЕ: Если у вас возникли проблемы с тем, что специальные символы не отображаются:
- Убедитесь, что ваши файлы рецептов сохранены как UTF-8 и
- Включите следующий флаг: "-Dfile.encoding = UTF-8" в сценарий запуска..

Шансы должны быть в сумме 100%. Если вы не определите вероятность для одного или нескольких предметов, шанс будет равномерно рассчитан для оставшегося шанса до 100%.

Названия рецептов

Названия рецептов используются в книгах рецептов.

Если вы определяете собственное имя, оно должно быть уникальным и содержать от 3 до 32 символов (рекомендуется 18).

Пользовательское имя будет отображаться в книгах, если оно определено, иначе будет использоваться результат.

В'books/*.yml' files имена рецептов (пользовательские или автоматически созданные) используются для добавления и сортировки рецептов в книгах.

Рецепт предмета

Рецепты предметов - это предопределенные рецепты, которые вы можете использовать вместо предметов / результатов в некоторых местах.

В отличие от большинства рецептов, название рецепта требуется, чтобы на него можно было ссылаться позже.

Формат:

ITEM <recipe name> = <material:[data]:[amount]> [@ result flags]

Использование:

Когда упоминается, например, в @BundleItem, формат будет:

@bundle item:<recipe name>

Синтаксис рецепта

Рецепт создания / фигурного верстака

Формат шаблона
Простой формат
CRAFT [recipe name] [@ recipe flags] [group name] pattern abc | def | ghi <letter> <material:[data][, ...]> [@ ingredient flags] [... more ingredients] = [chance]% <material:[data]:[amount]> [@ result flags] [... more results]

[Имя группы] определяет группу книги знаний, к которой принадлежит этот рецепт.

Линия узора определяет форму сетки рецептов.

  • Каждая строка отделяется вертикальной чертой: |
  • Каждая буква представляет один или несколько ингредиентов, которые определены в разделе ингредиентов.
  • Буквы могут быть в любом порядке и могут быть любыми допустимыми символами от a до z.
  • Примеры:
    • abc | def | ghi = полная сетка 3x3
    • a | b | c = три высоких столбца ингредиентов
    • z | q | r = то же, что и выше, но с другими буквами
    • ab = два широких рецепта
    • aaa | bbb | ccc = Рецепт 3x3 с одинаковыми ингредиентами в каждом ряду

<буква> должна быть буквой, которая определена в строке шаблона выше

Ингредиенты могут быть определены материалом с дополнительными данными.

[@ флаги ингредиентов] поддерживает любые флаги, которые напрямую изменяют ингредиент. Это позволяет настраивать ингредиенты.

Несколько ингредиентов можно определить двумя способами:

  1. Добавление запятой после первого материала: данные, например: "iron_sword:2, cobblestone"
  2. Добавление еще одного объявления материала < письма>:данных, позволяющих использовать разные ингредиенты, включая метаданные флага, для одного и того же рецепта.

 

Что касается результата 'chance%' часть является необязательной, и если она определена, тогда "шанс" должен быть числом от 0,0 до 100,0 (да, принимает десятичные дроби)

Часть "[...]" означает, что вы можете повторить предыдущий оператор, поэтому вы можете определить более одного результата, и рецепт случайным образом выберет один из результатов.

Вы можете не указывать случайное число или использовать *% даже для его вычисления.

Вы также можете добавить к рецепту шанс неудачи, указав в качестве одного из результатов fail, air или 0.

Формат шаблона
Простой формат
COMBINE [recipe name] [@ recipe flags] [group name] pattern abcdefghi <letter> <material:[data][, ...]> [@ ingredient flags] [... more ingredients] = [chance]% <material:[data]:[amount]> [@ result flags] [... more results]

В отличие от рецепта формы, ингредиенты этого рецепта можно расположить в любом порядке в любом месте сетки.

[group name] определяет группу книги знаний, к которой принадлежит этот рецепт.

  • Каждая буква представляет один или несколько ингредиентов, которые определены в разделе ингредиентов.
  • Поддерживается до 9 букв для заполнения сетки 3x3
  • Буквы могут быть в любом порядке и могут быть любыми допустимыми символами от a до z.
  • Примеры:
    • abcdefghi = полная сетка 3x3
    • abc = три ингредиента
    • zqr = то же, что и выше, но с другими буквами

Рецепт плавки / печи

SMELT [recipe name] [@ recipe flags] [group name] [xp amount] <material:[data][, ...]> % [time] [@ ingredient flags] [... more ingredients] & [material:[data]] = [chance]% <material:[data]:[amount]> [@ result flags] // one result max!

[group name] определяет группу книги знаний, к которой принадлежит этот рецепт.

[xp amount] определяет количество опыта, которое дается за рецепт печи. Может быть десятичным числом. Примеры ("xp 2", "xp .5", "xp 1.1"). По умолчанию 0.

Первый '<material[, ...]>' это ингредиент.

(по желанию) '% [time]' может быть временемем плавки, 'time' должно быть 'instant', числом с плавающей запятой или диапазоном чисел с плавающей запятой, разделенных тире. Время по умолчанию: 10

(по желанию) '& [material:[data]]' может служить ЕДИНСТВЕННЫМ топливом для этого рецепта (рецепт топлива для этого предмета не требуется!)

И '= [chance]% <material:[data]:[amount]>' является результатом с необязательным шансом на успех, но этот тип рецепта поддерживает только 1 результат!

Рецепт топлива

FUEL [recipe name] <material:data> % <time> ...

Добавляет топливо с указанным временем горения.

'time' является обязательным и может быть числом с плавающей запятой или диапазоном чисел с плавающей запятой, разделенных тире.

Часть "..." означает, что вы можете указать более одного элемента, все они добавляются так же, как отдельные рецепты топлива.

Рецепты костра

CAMPFIRE [recipe name] [@ recipe flags] [group name] [xp amount] <material[, ...]>:[data] % [time] [@ ingredient flags] [... more ingredients] = [chance]% <material:[data]:[amount]> [@ result flags]

[group name] определяет группу книги знаний, к которой принадлежит этот рецепт.

[xp amount] определяет количество опыта, которое дается за рецепт печи. Может быть десятичным числом. Примеры ("xp 2", "xp .5", "xp 1.1").По умолчанию 0.

Первый '<material[, ...]>' это ингредиент.

(по желанию) '% [time]' может быть временем для приготовления, 'time' должен быть'instant', числом с плавающей запятой или диапазоном чисел с плавающей запятой, разделенных тире. Время по умолчанию: 30

NOTE: Due to API limitations, random cook times will only be set random on server start for campfire recipes.

И '= [chance]% <material:[data]:[amount]>' результат с необязательным шансом на успех.

Компост / Рецепты Компостницы

COMPOST [recipe name] [@ recipe flags] <material[, ...]>:[data] % [chance per level] % [levels] [@ ingredient flags] [... more ingredients] = [chance]% <material:[data]:[amount]> [@ result flags]

Первый '<material[, ...]>' это ингредиент.

(по желанию) '% [chance per level]' это процентная вероятность того, что уровень будет повышен на [levels]. Значение может быть числом с плавающей запятой >0 и <=100.По умолчанию 100.

(по желанию) '% [levels]' уровень, который нужно добавить в компостер. Значение может быть числом с плавающей запятой >0 и <=7. о умолчанию 1.

Пока компостер используется для рецепта, он будет привязан к этому рецепту (или к тому же результату из другого рецепта), поэтому конфликтующие рецепты невозможны.

Удаление рецепта

Рецепты извлечения ванилы

Используйте команду /rmextract чтобы экспортировать все рецепты ванили.

Файл будет создан в "/plugins/RecipeManager/extracted/" (для устаревших версий: "/plugins/RecipeManager/recipes/disabled/")

Откройте сгенерированный файл в текстовом редакторе и найдите рецепт, который вы хотите удалить.

Например, железный меч должен выглядеть примерно так:

craft iron_ingot iron_ingot stick = iron_sword:0:1

Найдя рецепты, которые вы хотите удалить, скопируйте их в файл в "/plugins/RecipeManager/recipes/"

Когда рецепты будут в вашем новом файле, добавьте @remove в соответствии с Remove Syntax ниже.

Удалить синтаксис

Чтобы удалить отдельный рецепт, поместите @remove сразу после строки craft/combine/smelt line:

craft @remove iron_ingot iron_ingot stick = iron_sword:0:1

Поместите @remove вверху файла, чтобы удалить все рецепты в файле.

@remove craft iron_ingot iron_ingot stick = iron_sword:0:1 [... more recipes]

Дополнительные параметры настройки и примеры см. На страницеadvanced recipes.html.
